# Working Agreements (Stef + Hermes)

## Scope
- This repository is the shared workspace for planning, docs, scripts, and project context.
- Use `projects/` for project-specific work.

## Structure
- `projects/` active project folders
- `docs/` shared notes, decisions, SOPs
- `scripts/` automation helpers
- `assets/` images and reference files

## Commit style
- `feat:` new capability
- `fix:` bug fix
- `chore:` maintenance/refactor/docs updates

## Safety
- Never commit secrets (`.env`, API keys, tokens, private certs).
- Keep sensitive business material private unless explicitly marked for public sharing.

## Collaboration defaults
- Prefer small, clear commits.
- Keep README/docs current when structure changes.
- Use bullet-list summaries in notes and status updates.
